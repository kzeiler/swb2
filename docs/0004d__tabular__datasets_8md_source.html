1 Tabular data {#tabular_datasets}
2 ----------------------------------
3 
4 In addition to the gridded data, one or more lookup tables must be
5 provided in order to supply parameter values to the SWB modules. Many
6 parameters are specified for specific combinations of landuse
7 categories
8 and hydrologic soils groups. A list of the required parameters for
9 each
10 SWB module may be found in the appendices.

43 
44 In the original version of the SWB model, the parameters given above
45 were “hard-wired”; in other words, SWB required the lookup table to be
46 structured such that the parameters were supplied in a non-flexible
47 column order. In this way, parameter values for curve numbers would be
48 supplied in the first columns, followed by the maximum recharge rates,
49 interception values, and rooting depths. In addition, the original
50 version of SWB required that the number of soil types and landuses be
51 specified, and did not allow nor require a table header.
52 
53 The new version of SWB uses keywords to identify parameter values
54 within
55 the table; the new lookup tables allow parameters to be supplied in
56 any
57 arbitrary column order. A separate column of parameter values must be
58 supplied for each soil type. A snippet of the new table format is
59 given
60 below.
