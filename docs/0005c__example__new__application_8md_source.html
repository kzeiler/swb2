1 Example Setup of a New SWB Application {#example_new_application}
2 ====================================================================
3 
4 [TOC]
5 
6 This section presents development of a complete SWB 2.0 application
7 for
8 the state of Wisconsin. The tasks involved in setting up a new SWB
9 application are:
10 
11 1. Determine geographic area of application;
12 2. Decide on projection system (if any) to be used;
13 3. Determine which process options within SWB are most appropriate;
14 4. Locate and possibly download the required climate and GIS datasets;
15 5. Perform GIS operations, especially if flow routing is to be used;
16 6. Examine the metadata for each of the climate and GIS datasets, and
17  add directives to the control file as appropriate.
18 7. Create one or more lookup tables to supply all needed parameters
19  for
20  the process options in use.
21 
22 Geographic Area of Application
23 ==============================
24 
25 For this example we're targeting the entire state of Wisconsin, USA.

28 Climate Data
29 ============
30 
31 New sources of gridded climate data come online every day. Many of
32 these
33 gridded datasets can be used to provide SWB with the needed
34 precipitation and air temperature data. Making use of these new
35 gridded
36 datasets, however, requires the user to research the ways in which
37 these
38 new data were encoded into the netCDF file format. In this section we
39 document the steps taken in order to generate the control file
40 statements needed to make SWB understand a new data source.
41 
42 The Global Precipitation Climatology Project (GPCP) generates
43 estimates
44 of precipitation for the entire globe on a daily basis by merging
45 microwave, infrared, and sounder datasets with precipitation gage
46 data.
47 The resolution is 1-degree by 1-degree, far too coarse for many SWB
48 applications, but entirely suitable for this example. The datasets may
49 be found at
50 <https://climatedataguide.ucar.edu/climate-data/gpcp-daily-global-precipitation-climatology-project>.
51 The remainder of this example assumes that the new netCDF datasets ave
52 been downloaded and placed in a local subdirectory, and that the
53 required software drivers for netCDF files have been downloaded and
54 installed on the user's machine. The netCDF C libraries and utility
55 programs demonstrated in this example may be obtained [here]
56 (<http://www.unidata.ucar.edu/software/netcdf/docs/winbin.html>).
57 
58 The first step to take when attempting to use a new data source is to
59 determine the structure, naming conventions, and geographic projection
60 used in packaging the data into the netCDF file format. The program
61 "ncdump" is a utility distributed along with the netCDF C language
62 library. `ncdump -h` will comb through the netCDF file and return only
63 header information:
64 
65  ncdump -h GPCP_1DD_v1.2_201306.nc
66
